Widget Settings

By default, BiovizJS is loading an empty 3D scene when you deploy it to your website. You may decide to start BiovizJS with a specific context as :

  • a loaded structure
  • a loaded script
  • ...

To do so, you have to use the available deploy options when deploying a BiovizJS widget. The different deployment options are described as following :

  • biovizPath : string
    Path of bioviz.html file containing the application (by default, it's path is ./biovizJS/bioviz.html)

  • defaultRepresentations : boolean Enabled by default, will set bioviz to automatically show default representations of each parts of a structure.

  • structureToLoad : string
    The structure to load at start
    You can either set a structure ID or a structure URL. When defining a structure ID, it will try to catch it on the RCSB PDB database or the customUrl.

  • inputScript : string
    The script to load at start

  • startScript : boolean
    Start the script or not when loading is finished

  • enableScriptPlayer : boolean Enable or not the BiovizJS script player

  • pdbCustomUrl : string
    The URL has to be like the following example :
    http://www.rcsb.org/pdb/files/${id}.pdb
    (${id} will be replaced by Bioviz when choosing a structure to load)
    You can also add a ${divided} string on the URL, it will be used for the directory name of the database.
    If you specified pdbCustomUrl, you can't specify pdbxCustomUrl.

  • pdbxCustomUrl : string
    Custom URL for PDBX files access
    The URL has to be like the following example :
    http://www.rcsb.org/pdb/files/${id}.cif
    (${id} will be replaced by Bioviz when choosing a structure to load)
    You can also add a ${divided} string on the URL, it will be used for the directory name of the database.
    If you specified pdbxCustomUrl, you can't specify pdbCustomUrl.

  • advancedRepresentation : boolean
    Enable advanced representations as Alpha Shape (not implemented yet)

  • background : string
    Background of the scene in hexadecimal string ('#000000')
